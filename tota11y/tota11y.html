Version 2.6 ("Psychopathic Tapeworm")

  • Contrast checker now ignores transparent (therefore, invisible) text, eg on Amazon, Guardian

Version 2.5 ("Concupiscent Cormorant")

  • Empty elements plugin: show yellow warning for empty p and multiple br elements, as they may indicate shonky CSS but aren't a disaster for a11y
  • Add tests for empty nav, header, main, aside, footer, figcaption. These will be announced to screen reader users and make them furiously grumpy.
  • When hovering over a tota11y label, bump its z-index in case it is obscured by a nearby label in busy pages.
  • Make the menu responsive when screen is zoomed, as discovered by Taylar.

Version 2.4 ("Insouciant hedgehog")

  • A criminally ugly hack to suppress reporting contrast violations if text is visually hidden using the common clip pattern which we use on Babylon. This is to reduce false positives which erode trust in the test.

Version 2.3 ("Sultry panda")

  • Fixed clean-up bug in new Titles module
  • New Titles module to show missing titles on iframes, and superfluous titles on other things erroneously put there to placate the false god of Search Engine Optimists

Verson 2.2 ("Mellifluous ostrich")

  • Correct calculation of contrast ratio in the contrast module to take account of boldness of text and not just rely on font-size. (Thanks Mozilla dev tools!)

Version 2.1 ("Uninhibited gibbon")

  • New module to highlight empty elements, eg heading and paragraphs, which was a constant bugbear on US marketing site
  • Tweaked img/alt module to ask users to check accuracy of alt text (rather than perhaps falsely-reassure that presence of alt text is actually useful)

Version 2.076303987 ("Obstreperous rhinocerous")

  • Add links to Babylon DNA guidance where applicable
  • Split out modules into 'designers'/ content editors and Developers (was 'Experimental')

Version 2 ("Lachrymose Lemur")

  • New module to expose HTML5 landmarks (footer, header etc) and explicitly set ARIA roles (but not those that are implicit, because that's not as useful for diagnosing coder errors)
  • Tweaked name and description module to stop referring to 'screen readers'; added exposure of value attribute on input type=submit fields (as that is what gets exposed and wasn't being reported). Add value of aria-describedby attributes as that is also passed to AT, especially as hints/ instructions on form fields
